Panel B · Memory

Four tiers, by lifespan.

Organised by how long things live, not by topic. Index pattern keeps always-loaded context to ~1–2 KB while making thousands of memories available on demand.

B2 · Hunch → heuristic

3-confirmation rule
  1. 1×
    Observation loggedNoted by collector. Not yet in memory.
  2. 2×
    Hunch tentativeAdded to working-memory, labelled.
  3. 3×
    Heuristic permanentPromoted to Tier 2. Applied going forward.
  4. ↺
    Contradicted demotedReturns to hunch, or archived with note.

Each Tier-2 entry stores why + how to apply — so the mind can judge edge cases instead of blindly following.

Panel C · Self-maintenance

Three concentric loops.

The mind curates itself while you work, not just at logoff. Catching corrections within minutes produces tighter rules than catching them hours later.

Inner loop

Mid-session — every N tool calls

PostToolUse hook
Collect signalsmemory-collector.py
Every ~10 calls: scans live transcript, appends to working-memory.md while context is hot.
correctionconfirmationfollow-throughfrustration
Promote / demotememory-curator.py
Every ~25 calls: applies the 3-confirmation rule. Hunch → heuristic. Contradicted → demoted. Stale (>5d) → archived.
Middle loop

Per session — bookends

SessionStart · SessionEnd
Session startsession-start-health.py
Loads identity + MEMORY.md + working-memory. Warns if curator hasn't run in >7 days.
Session endsession-metrics.py
Writes one row to session-metrics.csv. Heavy maintenance already happened in the inner loop — this is the safety net.
Outer loop

Weekly — time-gated

7-day timestamp
Compute hypothesesmetrics-brain.py
Reads CSV, computes H1–H5, writes daily-report.md. Measurement keeps the system honest.
Regenerate indexmemory-index-weekly.py
Optional. Rebuilds memory/INDEX.md grouped by rules + decisions.

Why nest them? By the time a session ends, the model has drifted from the moment of insight. The inner loop catches corrections within minutes; bookends and weekly review are safety nets and measurement.

Panel D · Findings

Does it actually help?

9,869 sessions over 3 months. Matched cohort: 71 with memory vs 21 without (≥30 tool calls, post-instrumentation).

Memory-loaded sessions are denser and orient faster. But H5 — the headline outcome (fewer repeated corrections) — came back null. We don't have a clean A/B yet. Publishing specifically to get more data and pushback on experimental design.
