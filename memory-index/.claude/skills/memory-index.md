# memory-index

Interactive wizard for the MindMemory memory-index feature. Configures sources, registers the SessionStart hook, runs the generator, and shows the result.

## When to invoke

- User says "memory-index", "/memory-index", "set up the index", "configure memory index"
- User says "regenerate the index", "rerun memory index", "show me the index"
- User asks "is the memory index running?"

## Workflow

### Mode detection
First, detect what the user wants:

1. **Setup** — `config/memory_index.config.json` does NOT exist → run setup wizard
2. **Run now** — config exists, user said "regenerate" / "run" / "rerun" → execute generator
3. **Status** — user asked "is it running?" / "show status" → report state

### Setup wizard

Walk the user through these questions, accepting Enter for defaults:

1. **Where are your rules files?**
   - Default: `memory/rules`
   - Each `.md` file in this folder is treated as one rule. Title comes from the first `# Heading`.

2. **Where is your working memory file?**
   - Default: `memory/working-memory.md`
   - The generator extracts bullets from a `### Key Decisions` section.

3. **What is the section heading for decisions?**
   - Default: `Key Decisions`

4. **Where should INDEX.md be written?**
   - Default: `memory/INDEX.md`

5. **Register the SessionStart hook?**
   - Default: yes
   - If yes: append the hook entry to `.claude/settings.json`. If `settings.json` does not exist, copy `settings.json.example`.
   - The hook gates by a 7-day timestamp file at `memory/.memory_index_last_run` and spawns the generator in the background.

6. **Run a first-time generation now?**
   - Default: yes

After confirmation:
- Write `config/memory_index.config.json` with the chosen paths
- Register the hook if requested
- Run `python tools/memory_index_generate.py --dry-run` and show output to user
- If user approves, run without `--dry-run` and confirm INDEX.md location

### Run now

Execute: `python tools/memory_index_generate.py`
Read the resulting `INDEX.md` and show the user a concise summary (count of rules, count of decisions, output path).

### Status

Report:
- Config file present? Path?
- Hook registered in `.claude/settings.json`? (grep for `memory-index-weekly`)
- Last run timestamp (mtime of `memory/.memory_index_last_run`)
- INDEX.md present? When was it last updated?
- Next scheduled regeneration (last_run + 7 days)

## Rules

- Never hand-edit `INDEX.md` — it is fully regenerable. If output is wrong, fix the source files or `config/memory_index.config.json`, then re-run.
- The hook never fails the session. All errors swallowed by design.
- The generator never makes network calls or LLM calls — pure file IO.
- If a user complains about output, run with `--dry-run` first to inspect before overwriting.

## Files this skill touches

| File | Action |
|------|--------|
| `config/memory_index.config.json` | Created during setup |
| `.claude/settings.json` | Hook entry appended during setup |
| `memory/INDEX.md` | Generated/regenerated by the script |
| `memory/.memory_index_last_run` | Touched after each regeneration |
